## 3. 模块依赖关系

### 3.1 依赖层级图

```
Layer 1: 基础服务层 (无依赖)
├── Auth (认证授权) ──────────────────┐
├── User (用户管理) ──────────────────┤
├── Finance (钱包系统) ────────────────┤
├── System (系统管理) ────────────────┤
└── Shared (共享服务) ────────────────┤
                                        │
Layer 2: 业务服务层 (依赖Layer 1)     │
├── Writer (写作创作) ────────────────┤
├── Reader (阅读器) ──────────────────┤
├── Bookstore (书城) ─────────────────┤
├── Social (社交互动) ────────────────┤
└── Audit (内容审核) ─────────────────┤
                                        │
Layer 3: 增值服务层 (依赖Layer 1+2)   │
├── AI (AI服务) ──────────────────────┤
├── Recommendation (推荐系统) ─────────┤
├── Search (搜索系统) ─────────────────┤
└── Stats (数据统计) ─────────────────┤
                                        │
```

### 3.2 核心依赖关系

#### 3.2.1 Auth (认证授权) → 所有模块

**依赖类型**: 基础依赖
**被依赖模块**: Writer, Reader, Bookstore, Social, AI, Admin等
**依赖内容**:
- 用户身份验证
- JWT令牌解析
- RBAC权限控制

#### 3.2.2 Writer (写作创作) ↔ Bookstore (书城)

**依赖类型**: 双向依赖
**依赖关系**:
- Writer → Bookstore: 发布作品到书城
- Bookstore → Writer: 获取作品信息
**集成点**:
- `PublishService` (writer/publish_service.go)
- `BookstoreService` (bookstore/bookstore_service.go)

#### 3.2.3 Reader (阅读器) → Bookstore (书城)

**依赖类型**: 单向依赖
**依赖内容**:
- 获取书籍和章节信息
- 章节付费验证
- VIP权限验证

#### 3.2.4 AI (AI服务) → Writer (写作创作)

**依赖类型**: 单向依赖
**依赖内容**:
- 获取项目上下文
- 调用写作工具（大纲、角色卡、时间线）
- 文档内容向量化（RAG）

#### 3.2.5 Social (社交互动) → Writer + Reader + Bookstore

**依赖类型**: 多向依赖
**依赖内容**:
- 评论对象（书籍、章节）
- 点赞对象（书籍、章节、评论）
- 关注对象（作者）

#### 3.2.6 Finance (钱包系统) → Reader + Writer

**依赖类型**: 反向依赖
**依赖关系**:
- Reader → Finance: 章节付费、会员购买
- Writer → Finance: 稿费结算、收入提现
**集成点**:
- `ChapterPurchaseService` (bookstore/chapter_purchase_service.go)
- `AuthorRevenueService` (finance/author_revenue_service.go)

#### 3.2.7 Recommendation (推荐系统) → Reader + Bookstore

**依赖类型**: 数据依赖
**依赖内容**:
- 阅读历史数据
- 用户行为数据
- 书籍元数据

### 附录C: 问题优先级定义

#### P0 - 阻塞性问题
- 影响核心功能
- 用户体验严重受损
- 必须立即解决

#### P1 - 高优先级问题
- 影响重要功能
- 用户体验受损
- 1-2周内解决

#### P2 - 中优先级问题
- 影响次要功能
- 用户体验略有影响
- 1-2月内解决

#### P3 - 低优先级问题
- 功能增强
- 不影响当前使用
- 长期规划

### 附录D: 合规性评级标准

#### ✅ 优秀 (90%+)
- 设计文档完整
- 实现完整
- 测试覆盖良好
- 无严重问题

#### 🟡 良好 (70%-89%)
- 设计文档基本完整
- 实现基本完整
- 测试覆盖一般
- 有少量问题

#### 🟡 中等 (50%-69%)
- 设计文档部分缺失
- 实现部分缺失
- 测试覆盖不足
- 有较多问题

#### 🔴 差 (<50%)
- 设计文档严重缺失
- 实现严重缺失
- 测试覆盖不足
- 有严重问题
