## 🎯 设计原则

### 单一职责原则

每个模块只负责一个明确的功能领域，避免职责混乱。

### 依赖倒置原则

高层模块不依赖低层模块，通过接口进行解耦。

### 开闭原则

对扩展开放，对修改关闭，通过抽象和接口实现灵活扩展。
